

**把程序启动脚本放到 /etc/init.d/ 目录下，名字以 Sxx 开头，就会开机自动运行！**

xx 是数字，数字越小启动越早。

---

## 第一步：创建一个启动脚本

在 `/etc/init.d/` 下面创建一个文件，比如：

`S99myapp`

内容**直接复制我下面这段**，把里面的 **/path/to/your/app** 改成你程序的真实路径：

sh

```
#!/bin/sh

case "$1" in
    start)
        echo "Starting my application..."
        /path/to/your/app &
        ;;
    stop)
        echo "Stopping my application..."
        killall your-app-name
        ;;
    restart)
        $0 stop
        $0 start
        ;;
    *)
        echo "Usage: $0 {start|stop|restart}"
        exit 1
        ;;
esac
```

### 必须改的地方：

1. `/path/to/your/app` → 你程序的真实路径
    
    例如：`/usr/bin/myapp`
2. `your-app-name` → 你程序的名字（用于关闭）

---

## 第二步：给脚本加可执行权限（必须做）

sh

```
chmod +x /etc/init.d/S99myapp
```

---

## 第三步：完成！已经可以开机自启了

因为你的系统有 **[[rcS文件]]** 自动执行所有 S 开头脚本，

所以**不需要任何额外配置**，重启就会自动运行！

---

# 最简单的极简版（如果你不想写复杂脚本）

如果你只想**最简单开机运行**，直接创建脚本 `S99myapp`：

sh

```
#!/bin/sh
/path/to/your/app &
```

然后加权限：

sh

```
chmod +x /etc/init.d/S99myapp
```

**重启就会开机自启！**

---

# 启动顺序怎么选？（S 后面的数字）

- S00~S20：系统底层（挂载、驱动）
- S30~S60：系统服务（WiFi、蓝牙、网络）
- **S90~S99**：你的应用程序（推荐用 S99）

---

# 测试是否成功（不用重启）

sh

```

```

能运行就代表成功！
