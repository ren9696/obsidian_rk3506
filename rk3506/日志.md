我用**嵌入式 Linux 最实用、最接地气**的方式，一次性把「日志」讲透，包括概念、知识点、以及它和 Shell、C 程序的关系，你看完就能直接用在项目里。

# 一、日志是什么？

一句话：

**日志 = 程序 / 系统运行时，把 “发生了什么” 按时间顺序记录下来的文本文件。**

就像飞机黑匣子、行车记录仪。

程序跑着跑着，把关键信息写进文件里，方便你事后看：

- 程序什么时候启动的
- 正常执行到哪一步
- 哪里出错了
- 网络通不通
- 传感器数据是否异常
- 系统资源够不够

日志一般就是普通文本文件，比如：

- `/var/log/messages`
- `/tmp/app.log`
- `/var/log/syslog`

# 二、日志有哪些核心知识点？

## 1. 日志级别（

程序不会什么都记，会分级：

- **DEBUG**：调试信息，开发时用
- **INFO**：正常流程信息
- **WARN**：警告，不影响运行但要注意
- **ERROR**：错误，功能异常
- **FATAL**：致命错误，程序要挂了

## 2. 日志格式

一条标准日志一般包含：

`时间 + 级别 + 进程ID + 模块 + 内容`

例如：

plaintext

```
2026-04-09 15:30:01 [INFO] [1234] [MAIN] 程序启动成功
2026-04-09 15:30:02 [ERROR] [1234] [UART] 串口打开失败
```

## 3. 日志输出位置

- 输出到**文件**（最常用）
- 输出到**串口 / 控制台**
- 输出到**系统日志服务**（syslog、rsyslog）
- 远程上传到服务器

## 4. 日志切割（log rotate）

日志会越写越大，占满 Flash/EMMC，所以必须：

- 按大小切割
- 按天切割
- 自动删除旧日志

嵌入式里这一步**基本都靠 Shell 脚本做**。

## 5. 日志等级控制

运行时可以开关日志：

- 产品发布只开 INFO/ERROR
- 调试时开 DEBUG

## 6. 日志与系统稳定性

- 日志不能写太频繁，否则占 CPU、Flash 寿命
- 不能在中断里疯狂打日志
- 不能无限制写，必须限制大小

# 三、日志与 C 程序、Shell 脚本 的关系

这是你最关心的，我讲得非常直白：

---

## 1. C 程序：产生日志的主体

你的业务逻辑、硬件操作、协议通信，**所有关键信息都由 C 程序输出日志**。

C 里打日志常见方式：

- 简单版：`fprintf(log_file, "...");`
- 标准库：`syslog()`
- 自己封装日志库（带级别、时间、文件名行号）

C 程序是**日志生产者**。

---

## 2. Shell 脚本：管理日志的主力

Shell 不怎么产生业务日志，但**全权负责日志的维护**：

Shell 对日志做这些事：

1. **启动 C 程序时，把程序日志重定向到文件**
    
    sh
    
    ```
    ./my_app >> /var/log/app.log 2>&1
    ```
    
2. **定时清理、切割日志**
    
    sh
    
    ```
    # 超过1M就备份并清空
    if [ $(du -k /var/log/app.log | awk '{print $1}') -gt 1024 ]; then
        mv /var/log/app.log /var/log/app.log.bak
        > /var/log/app.log
    fi
    ```
    
3. **查看、过滤日志**
    
    sh
    
    ```
    tail -f /var/log/app.log
    grep ERROR /var/log/app.log
    ```
    
4. **开机自动创建日志目录**
5. **上传日志到服务器**

**Shell 是日志管理者。**

---

## 3. 三者关系总结

plaintext

```
C 程序  ——产生→  日志内容
Shell脚本 ——管理→ 日志文件（保存、切割、清理、重定向）
你（开发者） ——查看→ 日志，定位问题
```

更形象一点：

- **C 程序是说话的人**
- **日志是说出来的话**
- **Shell 是负责把话记在本子上、整理本子的人**

# 四、嵌入式里最经典的日志工作流（你项目直接照搬）

1. C 程序里写日志函数，输出到 stdout/stderr
2. Shell 启动脚本把输出重定向到 log 文件
3. Shell 定时任务切割日志
4. 出问题时，你通过串口 / SSH 看日志定位 bug

示例：

### 1）C 程序输出日志

c

运行

```
printf("[INFO] 程序启动\n");
fprintf(stderr, "[ERROR] 传感器异常\n");
```

### 2）Shell 启动脚本保存日志

sh

```
#!/bin/sh
./my_app >> /tmp/app.log 2>&1 &
```

### 3）Shell 定时清理日志

sh

```
#!/bin/sh
# 日志大于1M则备份
find /tmp/ -name app.log -size +1M -exec mv {} app.log.bak \;
```

# 五、一句话总结

- **日志 = 程序运行的 “历史记录”**
- **C 程序负责 “打出日志”**
- **Shell 脚本负责 “存日志、切日志、清日志”**
- 嵌入式 Linux 项目里，**日志 + Shell + C 是铁三角**，缺一不可
