### `access`系统调用（通用文件存在性判断）

代码中在`config_file_recover`（配置文件恢复）等函数中，使用 **`access`系统调用 ** 判断文件（含配置文件）是否存在，属于 Linux 系统通用的文件检查方式。

#### 实现逻辑

调用`access(文件路径, F_OK)`，其中`F_OK`是系统宏，**仅用于检查文件是否存在**（不判断权限）：

- 返回`0`：文件存在；
- 返回`-1`：文件不存在（可通过`perror`查看具体错误）。

#### 代码中的使用示例

c

运行

```
// 检查正式配置文件是否存在
if (access(config->config_file_path, F_OK) == 0){
    // 存在则执行后续逻辑
}
// 检查备份配置文件是否存在
if (access(config->config_file_path_backup, F_OK) == 0){
    // 存在则执行后续逻辑
}
```
使用`access`函数**需要包含头文件**，核心必备头文件为`<unistd.h>`，同时该头文件也定义了`access`函数中用到的`F_OK/R_OK/W_OK/X_OK`等权限检查宏，无需额外引入其他头文件。

### 结合`config.c`代码的验证

在你提供的`config.c`代码中，**已经显式包含了`<unistd.h>`头文件**（代码第 16 行），这也是该文件中能正常调用`access`函数（如`config_file_recover`、`config_file_updata`函数中）的前提，代码片段佐证：

```
// 代码中已包含access的必备头文件
#include <unistd.h> 

// 代码中正常调用access，依赖<unistd.h>

```

### 补充说明

1. `access`函数的函数原型定义在`<unistd.h>`中：
    
    c
    
    运行
    
    ```
    int access(const char *pathname, int mode);
    ```
    
2. 函数中`mode`参数的常用宏（均在`<unistd.h>`中定义）：
    
    - `F_OK`：仅检查文件**是否存在**（代码中核心使用的宏）；
    - `R_OK`：检查是否有**读权限**；
    - `W_OK`：检查是否有**写权限**；
    - `X_OK`：检查是否有**执行权限**。
    
3. 该头文件是 Linux/Unix 系统下的标准头文件，跨平台（如 Windows）开发时无此头文件，也无`access`函数（Windows 下对应`_access`，头文件`<io.h>`）。
